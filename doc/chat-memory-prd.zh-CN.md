# Chat 记忆与用户偏好产品需求文档

[English](./chat-memory-prd.md)

> 产品基线：Workavera `0.0.8`。

## 1. 产品目的

Chat 记忆让 Workavera 的 AI Assistant 在用户控制下跨会话理解长期有效的个人信息。它补充会话历史和上下文压缩，但不替代其中任何一项。用户偏好用于保存控制记忆行为和界面外观的账户级私有设置。

这项能力透明且可逆：用户决定是否启用记忆、是否允许 Chat 自动生成记忆，以及每条记忆是否继续生效。

## 2. 目标

- 在不同 Chat 会话之间复用长期有效的用户事实和偏好。
- 明确区分短期会话上下文和长期记忆。
- 允许用户明确要求 Chat 记住或忘记信息。
- 可选地允许 Chat 自动保存长期价值较高的信息。
- 将每次 Chat 记忆修改展示为清晰、专用的界面结果。
- 允许用户在 Settings 中新增、搜索、编辑、启用、停用和删除记忆。
- 使用一条私有用户偏好记录保存外观和记忆控制项。
- 保持模型供应商无关和自托管能力，不依赖 embedding 或向量数据库。

## 3. 非目标

- 编辑或展示内部会话压缩摘要。
- 将完整会话、文档、工具输出或工作空间记录保存为记忆。
- 在首个版本中实现语义或向量检索。
- 每次回复后再发起一次模型请求提取记忆。
- 临时会话或会话级记忆模式。
- 在用户之间共享记忆，或通过 MCP 工具暴露记忆。
- 使用记忆替代 Board、Calendar、Contacts、Reading、Docs 或其他权威业务数据。

## 4. 记忆模型

Workavera 使用三个相互独立的上下文层：

| 层级 | 用途 | 用户可编辑 |
| --- | --- | --- |
| 最近消息 | 原样保留当前交流 | 否 |
| 会话摘要 | 压缩同一会话中的较早消息 | 否 |
| 长期记忆 | 跨会话复用长期用户信息 | 是 |

当前用户消息优先于已保存记忆。记忆始终被视为可能过时的数据，而不是指令。当前表达与记忆冲突时，Chat 遵循当前表达，并在获得权限时更新相应记忆。

## 5. 用户偏好

每位用户拥有一条私有 `user_preferences` 记录。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `owner` | relation → users | 必填、唯一，用户删除时级联删除 |
| `theme` | select | `system`、`light` 或 `dark` |
| `memory_enabled` | bool | 允许 Chat 读取和写入长期记忆 |
| `memory_auto_capture` | bool | 记忆启用时允许主动生成记忆 |
| `created`、`updated` | autodate | 记录时间 |

新账户默认使用 `theme=system`、`memory_enabled=false`、`memory_auto_capture=false`。

关闭记忆时保留 `memory_auto_capture` 的已保存选择，但自动生成的有效状态始终是 `memory_enabled && memory_auto_capture`。因此关闭记忆会停止 Chat 的全部记忆读取和写入，不删除数据，也不改变自动生成的保存选择。

系统级配置保存在 `configs`，账户级个人偏好保存在 `user_preferences`。

## 6. 长期记忆数据

长期记忆保存在 `chat_memories`。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `owner` | relation → users | 必填，级联删除 |
| `category` | select | `preference`、`personal`、`work`、`goal` 或 `constraint` |
| `content` | text | 单个长期事实，必填、去除首尾空白，最长 500 字符 |
| `active` | bool | 只有启用的记忆可供 Chat 使用 |
| `origin` | select | `manual`、`explicit` 或 `automatic` |
| `source_conversation` | relation → chat_conversations | 可选，来源删除时清空 |
| `source_message` | relation → chat_messages | 可选，来源删除时清空 |
| `created`、`updated` | autodate | 记录时间 |

分类含义：

- `preference`：沟通、格式、工具或工作方式偏好。
- `personal`：用户明确提供的稳定个人背景。
- `work`：长期有效的职业背景、约定或技术选择。
- `goal`：在未来会话中仍然有用的长期目标。
- `constraint`：持续生效的要求、限制或边界。

每个账户最多保存 50 条记忆。更新已有记忆不占用新名额；内容完全相同时不执行写入。

## 7. Chat 可以记住什么

Chat 只保存用户明确表达的信息。有效记忆必须简短、原子化、长期有效，并且可能改善未来会话。

Chat 不保存：

- 密码、API Key、访问 Token、认证材料或身份证件；
- 用户没有明确表达的推断或猜测；
- 临时任务状态、一次性请求或普通对话细节；
- 完整文档、大段摘录、完整消息或原始工具结果；
- 已由 Board、Calendar、Contacts、Reading 或 Docs 管理的信息。

新事实与已有记忆冲突时，Chat 更新相关记录，而不是新增重复项。删除记忆始终要求用户明确提出，或由用户直接在 Settings 中操作。

## 8. 记忆状态矩阵

| 记忆 | 自动生成 | 使用已有记忆 | 明确要求“记住” | 主动保存 |
| --- | --- | --- | --- | --- |
| 关闭 | 任意值 | 否 | 否，引导用户前往 Settings | 否 |
| 开启 | 关闭 | 是 | 是 | 否 |
| 开启 | 开启 | 是 | 是 | 是，但必须符合记忆策略 |

开启自动生成只代表持续授权符合策略的记忆写入，不授权修改 Workavera 的其他模块。

## 9. Chat 上下文与召回

每次运行按以下顺序组装上下文：

1. Workavera 基础系统规则和当前日期。
2. 当前公开用户资料。
3. 当前有效的记忆策略。
4. 全部启用的长期记忆。
5. 现有会话摘要和最近消息。

记忆以包含 ID、category、origin 和 content 的结构化条目提供。提示词明确说明记忆内容是不可信数据，不能覆盖系统规则或当前用户消息。

Saved Memories 区域包含本轮全部启用记忆，是当前记忆状态的权威来源。历史记忆工具调用和结果继续作为过去事件保留在会话历史和摘要中，但不能覆盖当前 Saved Memories。每个用户最多 50 条记忆，因此完整注入有明确上限，且不需要 embedding 基础设施。

偏好或记忆加载失败时，Chat 对记忆功能采用失败关闭，并在不使用长期记忆的情况下继续对话。

## 10. Assistant 工具

记忆工具只在已认证的 Chat 运行中提供，不进入 MCP 工具列表。

### `system_memory_upsert`

输入：

- 可选记忆 `id`，用于更新；
- 必填 `category`；
- 必填 `content`；
- 必填 `origin`，取值为 `explicit` 或 `automatic`。

服务端校验所有权、记忆数量上限、内容、分类、有效偏好状态和自动生成权限。来源会话和消息来自可信 Chat 运行，不接受模型输入。

结果返回 `created`、`updated` 或 `unchanged`、当前记忆，以及更新发生时的旧可编辑值。

### `system_memory_forget`

输入：

- 必填记忆 `id`。

工具只在用户明确要求遗忘时永久删除当前用户自己的记忆。结果标识被删除的记忆，供 Chat 展示清晰的完成状态。

## 11. Settings 体验

Settings 中包含 Memory 卡片：

- **Use memory in Chat**；
- **Automatically save useful details**；
- 记忆数量和 **Manage memories** 操作。

记忆关闭时，自动生成开关不可操作，但已保存值保持不变。

记忆管理器使用右侧 Sheet，并支持：

- 文本搜索和 category 筛选；
- 手工新增记忆；
- 编辑 category 和 content；
- 启用和停用；
- 查看 origin、时间和可用的来源会话链接；
- 单条删除；
- 经确认后删除全部记忆。

Chat 记忆关闭时，用户仍可管理已有记忆。

## 12. Chat 体验

记忆工具结果使用专用卡片，不展示通用工具 JSON。

- 明确要求保存时显示 **已按你的要求记住**。
- 自动保存时显示 **Chat 自动记住了这条信息**。
- 更新时明确说明已有记忆发生变化。
- 遗忘操作显示已完成状态。

新建和更新记忆支持 Undo。已认证的 Chat 接口验证所有权，并确认记忆仍与原工具结果一致；随后在同一事务中删除新建记忆或恢复更新前的可编辑值，同时将原工具输出持久化为 `undone`。记忆之后被修改时，Undo 失败且不会覆盖较新的状态。卡片同时提供记忆管理入口。

## 13. 权限与生命周期

- 偏好和记忆仅对 Owner 可见。
- 用户可以列出、查看和更新自己的偏好，但不能转移或删除偏好记录。
- 用户可以列出、查看、新增、更新和删除自己的记忆。
- 通过 API 手工新增时，服务端强制写入当前用户、`origin=manual`、启用状态和空 Chat 来源。
- 更新记忆时不能修改 owner、origin 或来源字段。
- 删除用户时同步删除其偏好和记忆。
- 删除来源会话或消息时保留记忆，并清空对应可选关系。

## 14. 验收标准

- 主题和记忆偏好按账户持久化，并且不作为系统配置暴露。
- 新老账户的记忆默认完全关闭。
- 三种记忆状态严格遵循状态矩阵。
- Chat 跨会话使用启用的记忆，同时不改变现有会话压缩行为。
- 当前用户表达覆盖冲突记忆。
- 显式和自动保存写入正确的 origin 与来源信息。
- 自动生成未有效开启时，服务端拒绝自动写入。
- 用户可以在 Settings 中新增、搜索、编辑、启用、停用、删除和清空记忆。
- Chat 记忆卡片准确展示新建、更新、无变化和遗忘结果。
- Undo 原子地撤销未发生后续修改的新建或更新，并将原工具结果持久化为 `undone`。
- Undo 不会覆盖原工具调用后发生变化的记忆。
- 每次运行的权威 Saved Memories 区域包含全部启用记忆。
- 用户不能读取或修改其他用户的偏好与记忆。
- MCP 不包含记忆工具。
- 主题选择和主题快捷键继续持久化，并且首屏不发生主题闪烁。
